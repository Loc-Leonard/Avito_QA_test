### TC-01 - Успешное создание объявления

**Цель:** проверить, что сервис корректно создает объявления при передаче валидного тела запроса.

**Шаги:**
1. Отправить запрос `POST /api/1/item` с телом:
```json
{
  "sellerID": 123456,
  "name": "Test item from Postman",
  "price": 1500,
  "statistics": {
    "likes": 1,
    "viewCount": 1,
    "contacts": 1
  }
}
```
2. Получить ответ от сервера.

**Ожидаемый результат**
- Статус ответа: `200 OK`
- В теле ответа присутствует поле `status` со строкой вида:
`Сохранили объявление - <идентификатор объявления>`

**Полученный результат**
```json
{
    "status": "Сохранили объявление - f67311ac-3b54-4e05-b071-907bf0cae584"
}
```

### TC-NF-01 - Отсутствие 5xx при валидных запросах

**Цель:** убедиться, что при отправке валидных запросов к API сервис не возвращает ошибки уровня сервера

**Шаги:**
1. Последовательно выполнить набор валидных запросов:
    - `POST /api/1/item` с корректным телом
    - `GET /api/1/item/:id` Для созданного объявления
    - `GET /api/1/statistic/:id` для получения статистики объявления
    - `GET /api/1/:sellerID/item` получить все объявления продавца
    - `DELETE /api/2/item/:id` удалить объявление
2. Зафиксировать статусы ответов

**Ожидаемый результат**
- Ни один из запросов не возвращает статусы `5xx`

**Полученный результат**
- Все запросы отвечают корректно, без ошибок со стороны сервера
