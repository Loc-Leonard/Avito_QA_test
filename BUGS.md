### BUG-01 - Сервис позволяет создать объявление с отрицательной ценой

**Шаги воспроизведения**
1. Отправить запрос `POST /api/1/item` с телом:
```json
{
  "sellerID": 985781,
  "name": "phone",
  "price": -1,
  "statistics": {
    "likes": 1,
    "viewCount": 2,
    "contacts": 3
  }
}
```
**Фактический результат**
- Статус `200 OK`
```json
{
    "status": "Сохранили объявление - 895362b8-b60f-4f81-808d-c3368328a6cc"
}
```

**Ожидаемый результат**
- Статус `400 Bad Request`
- В теле ответа сообщение о некорректной цене `(price не может быть меньше 0)`

**Серьезность**
Высокая - Нарушает базовую бизнес-валидацию, в систему попадают некорректные данные

### BUB-02 - Некорректное сообщение об ошибке при невалидном теле запроса
**Описание**
Для ряда невалидных запросов к POST /api/1/item сервис возвращает статус `"Не передано тело объявления"` и пустое поле `message`

**Шаги воспроизведения**
Отправить запрос `POST /api/1/item` с телом:
```json
{
  "sellerID": "lol",
  "name": "phone",
  "price": -1,
  "statistics": {
    "likes": 1,
    "viewCount": 2,
    "contacts": 3
  }
}
```
**Фактический результат**
- Статус `400 Bad Request`
```json
{
    "result": {
        "message": "",
        "messages": {}
    },
    "status": "не передано тело объявления"
}
```
**Ожидаемый результат**
- Статус ответа `400 Bad Request`
Тело ответа с более понятным описанием реальной ошибки (Неверный тип поля `sellerID`)

**Серьезность**
Средняя - валидация работает корректно, однако сообщение об ошибке не помогает клиенту диагностировать проблему

### BUG-03 - Несоответствие HTTP статуса и поля status при повторном удалении объявления
**Описание:**

При повторном запросе на удаление уже удаленного объявления HTTP-статус - `404 Not Found`, но в теле ответа в поле `status` содержится значение `500`, а сообщение пустое

**Шаги воспроизведения**
1. Создать объявление через `POST /api/1/item`
2. Отправить запрос `DELETE /api/2/item/:id` с полученным идентификатором, убедиться, что удаление успешно (Статус `200 OK`)
3. Повторно отправить запрос `DELETE /api/2/item/:id` с тем же ID
4. Дождаться ответа

**Фактический результат**
- HTTP статус `404 Not Found`
- тело ответа:
```json
{
    "result": {
        "message": "",
        "messages": null
    },
    "status": "500"
}
```

**Ожидаемый результат**
- поле `status` и HTTP-код согласованы `(Например "status": "404")`
- В поле `result.message` содержится понятное сообщение о том, что объявление не найдено или уже удалено

**Серьезность**
Средняя - Функционально сервис возвращает 404, но поле `status` и пустое тело ответа усложняет обработку ответа клиентом
